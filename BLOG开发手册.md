#### loadsh节流

#####滚动触发频率太高 需要做节流 / 防抖

​	防抖函数

​		触发间隔大于N的时候 执行

​		触发的时候不立刻执行函数  给一个计时 计时结束后执行函数  计时内如果再次触发 函数不会执行

​			监测用户不输入

​			短时间内大量触发同一个事件  指挥执行一次函数

​	节流函数   节流阀

​		函数拥有一个指定的执行间隔  N

​		短时间内大量触发事件行为  函数执行一次后  在N时间段内 不会工作  只有过了这个时间段  才会再次被调用

​			滚动条触发降频

​			每N时间内只能触发一次

#### 聊天室思路整理

```
0. 登录或者进入聊天室建立会话
1. 匿名和登录用户都可以使用聊天室
2. 登录用户重复登录会关闭会话
3. 用户退出聊天室不关闭会话

on
1. online 用户登录 ({uid,nikename})
  创建user项 uid nikname socket
    user[uid]={
      id:uid,
      nikename
    }
    socket.uid = uid
    socket.ghost = false
2. enterChat 进入聊天室 ({uid,nikename})
  1. 如果用户已登录 users[uid]
  2. 如果为游客 ({nikname})
    创建一个临时ID
    user[tempId] ={
      id:tempId,
      nikename
    }
    socket.uid = uid
    socket.ghost = true
3. leaveChat 退出聊天室
    1. 如果是游客
      socket.disconnect
    2. 如果是用户
      广播用户离开聊天室
4. 客户端断开连接
   delete users[socket.id]
```

精简的规则

#####优化一: wangeditor 使用cdn引入

​	引入了CDN之后 就不需要在打包wangeditor   就需要去调整一下打包内容 设置config.externals()方法去除不想打包的内容

#####优化二: lodash按需引入

​	lodash也改为cdn引入  因为没有使用太多东西  所以也可以不需要单独引用

​	按需引入  引入已经使用的内容  

​	使用的时候按需引入  需要使用两个对应的插件   可以在官网中找到

#####优化三: vue使用cdn引入  cdn分流

​	打包上线的时候  是使用运行时的vue版本   esm导入规则是按照es6的导入规则

​	平时打包的时候使用的版本是 vue.runtime.esm.js  

​	因为是在浏览器上使用  所以需要使用到浏览器版本   所以可以使用vue.runtime.min.js  里面包含了浏览器版本

#####优化四: 压缩代码

​	安装一个压缩插件  compression-webpack-plugin

​	需要让liveserver服务器开启gzip  才能在打包后使用压缩后的js文件

#####优化五: 路由的按需引用  路由懒加载

​	使用vuecli懒加载  

​	prefetch 静默加载 空闲的时候去加载资源  等用到的时候再加载 就算页面关闭了 也会完成当次加载

​	preload  预加载  提前加载的一种方式  页面一打开就开始加载 提前将要使用的资源先加载完

​	使用webpack的魔术注释 来导入我们需要的文件 异步导入  懒加载导入

​	什么时候需要一起打包进去  两个组件在用一个页面的时候

​	对路由使用懒加载 为了做到真正的使用的时候再加载  就需要停止掉prefetch
